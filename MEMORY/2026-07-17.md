(first-real Spotify client + backend schema/sync/routes + full drag-reorder frontend) — no single Robot run
(`ROBOT_MAX_TURNS=50`, `config.ts` → `robot/session.ts:119`) can finish it. Run 6 burned all 50 turns at
~1.1k tokens/turn (many small edits), killed mid-build → **no PR**. The C2 fault-tier would have promoted it
transient→deterministic (park `stuck`) at the 2nd identical signature.

queue FIRST: `PATCH status=prioritized` then `isEpic=true` — order matters, the epic guard bars queue lanes).

**Fix 2 — systemic (Steve: "the refine agent should know max turns is 50 and use it when decomposing").**

## Mac Mini migration (PD-188) — re-grilled step 3 against the post-Robot-loop reality

Steve asked LOE for the Mac Mini migration, then re-grilled **PD-188 step 3** (`/grill-with-docs`) — its
"Sortie egress stack lift-and-shift" text predated the D-055 C6/C7 Sortie retirement. Mini is confirmed set up.
Three genuine design decisions resolved (rest was a mechanical Sortie→agent-worker rename sweep):

1. **`/data` = Colima VM-native volume, NOT a macOS-host bind mount.** A host mount's virtiofs layer flattens
   Linux mode/uid semantics → would silently break the D-055 uid-split (mode-660 + `robot` uid-1500 exclusion is
   what kernel-enforces D-039). VM-native ext4 keeps it real, exactly as on the NAS. **Runtime-independent** — no
   macOS Docker runtime can preserve Linux ownership on a host mount. Trade-off: PD-190's off-box backup shipper
   must reach INTO the VM for `data/backups/`.
2. **Stay Colima for the cutover** (not OrbStack/native). Sortie's retirement touches none of D-035's Colima
   reasons; a Linux-VM runtime stays mandatory anyway (the egress squid-`internal:` jail + the uid-split are both
   Linux/Docker constructs — native macOS would drop both = security regression). Bonus: Colima's sudoless per-user
   docker FIXES the NAS's password-gated `sudo docker` (why the runaway PD-377 run couldn't be killed over SSH).
   PD-200's OrbStack re-eval stays on its ~1-month soak, decoupled from the migration.
3. **Disarmed bring-up.** Worker (now an autonomous coding agent) comes up dispatch-paused
   (`ROBOT_ALLOWLIST=NONE`/`ROBOT_DISPATCH_ENABLED=0`); verify egress + uid-split (`docker exec -u robot ... cat
   /data/dashboard.db` → Permission denied) + one refine turn on Colima; **arm only after** that passes AND the NAS
   loop is stopped (no double-loop vs the queue during the rollback window). Mirrors C1 prove-on-one.

Also: **no separate `.sortie.db`** (worker shares `dashboard.db`) and **no quota-refund cron** (C2 fault-tier
replaced Sortie's `max_sessions`; `ops/sortie/` gone). Worker is built **on-host** (not GHCR/Watchtower — D-044),
so the Mini needs the build toolchain. Board-URL env is now `ROBOT_BOARD_URL` (was `SORTIE_BOARD_URL`).

**Docs updated (uncommitted on `main`):** rewrote **PD-188 body step 3** via board PATCH (id 274, HTTP 200) +
