## PD-296 — watchdog now concurrency-aware for queued-stuck (PR #204, merged → completed)

The `sortie-watchdog.yml` queued sweep flipped any `sortie:queued` issue past the age threshold to `sortie:stuck` with **no concurrency awareness** — with `max_concurrent_agents:1`, one in-progress + N queued behind it falsely flagged the waiting queue. Fix: **skip the queued sweep entirely when `in-progress count >= limit`** (all slots busy → queue is expected, not stuck). Queued-stuck now fires only when a slot is free and work still sits queued (genuine non-dispatch / Sortie-down). In-progress sweep stays unconditional (a busy slot can still be a *stalled* one, caught by the 120m check, which then frees the slot next run).

- **Concurrency source = single source of truth:** parse `max_concurrent_agents` from `ops/sortie/WORKFLOW.md` at runtime (added an `actions/checkout` step — the job had none before; it only talks to GitHub via `gh`). Falls back to `1` (fails safe-closed) if the line can't be parsed. Chose parse-from-WORKFLOW.md over a mirrored env var specifically to avoid a drift bug (same class as the `agent-dashboard`→`task-monitor` rename).
- **`IN_PROGRESS_COUNT` snapshotted BEFORE the loop** — the in-progress sweep relabels stalled issues to stuck (freeing slots); the queued gate must reflect run-start state, not a slot this same run just vacated.
- **Cadence re-eval (Steve's call):** `*/15`→`*/30`, queued threshold `20m`→`30m` (cadence≈threshold per the file's own principle), in-progress 120m unchanged. ~halves run volume; queued-stuck is now a rare genuine-failure signal so minute-level detection isn't needed.
- Gotcha: `CONCURRENCY_LIMIT="$( grep… )"` under `set -euo pipefail` — a grep no-match (exit 1) in a command-substitution assignment aborts the script BEFORE the fallback `if` runs. Wrapped in `{ …; } || true`.
- Process: branched off `origin/main` in a worktree (concurrent session had `feat/site-status-sortie-worker` + dirty MEMORY/* checked out in the main tree); YAML-only so no npm dance. This is a **sensitive path** (CI workflow) but PD-308's path-guard isn't built yet, so nothing gated the merge.
