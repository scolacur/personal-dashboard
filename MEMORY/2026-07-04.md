## 20:06 — Worktree dev gotcha: vite `server.fs.allow` 403 kills hydration

**Symptom:** running `vite dev` from a git worktree serves SSR fine (routes return 200) but **every `onclick` is dead** — buttons "do nothing", `<a>`-based controls still navigate. Looks exactly like a component/hydration bug but isn't.

**Cause:** the worktree has no `node_modules` of its own, so Node resolves them from the **main repo** (outside the worktree). Vite's `server.fs.allow` sandbox then **403s the SvelteKit client entry** (`/@fs/…/node_modules/@sveltejs/kit/src/runtime/client/entry.js`) → the client module never loads → no hydration.

**Fix:** run `npm install` **inside the worktree** so `node_modules` lives within it. `npm run build` (adapter-static) and SSR are unaffected — no `@fs/` paths in prod, so this never shows up in the deployed app.

**Debug tooling note:** no playwright/puppeteer in the repo, but a ~90-line raw-CDP client (launch `Google Chrome --headless=new --remote-debugging-port`, drive via a hand-rolled WebSocket) proved the fix by clicking the toggle/flip and reading back DOM state + console errors. Kept in scratch, not committed.
